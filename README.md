## Overview

This repository analyzes on-chain behavior of two DeFi yield aggregator vaults on Ethereum:

- **Yearn USDC Vault** (`0xBe53A109B494E5c9f97b9Cd39Fe969BE68BF6204`): Analyzes deposit/redeem flows, strategy investment flows across lending protocols (Compound, Aave, Morpho, Sky), token transfer networks, and vault exchange rate development.
- **Cian Vault** (`0xB13aa2d0345b0439b064f26B82D8dCf3f508775d`): Analyzes multi-token flows (stETH, WETH, wstETH, ezETH, ETH) in USD, Aave leverage flows, token transfer networks, and vault exchange rate development.

## Data Collection

Transaction data was collected from an **archive Ethereum node** and token transfer data was collected via the **Etherscan API**. Exchange rate data was sourced from public price feeds. No proprietary or private data is included.

## Setup

### Prerequisites

- Python 3.9 or higher
- (Optional) Graphviz, for hierarchical network layouts:
  ```bash
  brew install graphviz       # macOS
  sudo apt install graphviz   # Ubuntu/Debian
  ```

### Installation

1. Clone the repository:
   ```bash
   git clone <repository-url>
   cd <repository-name>
   ```

2. Install Python dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. (Optional) Install `pygraphviz` for improved network layout quality:
   ```bash
   pip install pygraphviz
   ```

## Usage

### Python Notebooks

Open and run the notebooks in order. All notebooks use relative paths and expect to be run from the repository root.

```bash
jupyter notebook notebooks/yearn_analysis.ipynb
jupyter notebook notebooks/cian_analysis.ipynb
```

Each notebook is self-contained and will:
1. Load raw data from `data/`
2. Compute statistics and generate visualizations
3. Export processed results to `results/raw_data/`

### R Figures (ggplot)

Publication-quality figures are generated from `ggplot/ggplot2.Rmd`. It depends on the CSV exports produced by the Python notebooks above, so run those first.

**R package dependencies:**
```r
install.packages(c("ggplot2", "data.table", "dplyr", "magrittr",
                   "lubridate", "ggpubr", "ggrepel", "tidyr", "scales"))
```

Open `ggplot/ggplot2.Rmd` in RStudio and knit it, or render from the command line:

```bash
Rscript -e "rmarkdown::render('ggplot/ggplot2.Rmd')"
```

Output PDFs are saved to `ggplot/figures/`.

## Results

The `results/raw_data/` directory contains CSV exports of all intermediate and final data tables produced by the notebooks, including deposit/withdrawal time series, strategy flow data, protocol flow breakdowns, exchange rate series, and network node/edge tables.
